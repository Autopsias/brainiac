1 · Brand-new machine — macOS / Linux, from nothing

# 1. install the engine (tries uv → pipx → pip, first success wins; tells you which)
curl -fsSL https://raw.githubusercontent.com/Autopsias/brainiac/main/install.sh -o /tmp/brainiac-install.sh
bash /tmp/brainiac-install.sh

# 2. create your vault (pick any folder; created if missing)
BRAIN_VAULT=~/my-brain/vault brain init --full --apply

# 3. first search — works immediately, no model download needed yet
BRAIN_VAULT=~/my-brain/vault brain search "welcome" --json
That's it. Step 2, on an empty vault, scaffolds the folder structure, seeds three sample notes, and indexes them — so step 3 returns results right away (lexical mode). Semantic search arrives with the model in §5.

Make it permanent for a shell: export BRAIN_VAULT=~/my-brain/vault in your profile, then plain brain search "…" works everywhere.

2 · Brand-new machine — Windows (PowerShell, no WSL)

# 1. install (same channel chain; adds `brain` to your User PATH if needed)
irm https://raw.githubusercontent.com/Autopsias/brainiac/main/install.ps1 -OutFile install.ps1
.\install.ps1
# → if it says PATH was updated, open a NEW terminal before continuing

# 2. vault + first search
$env:BRAIN_VAULT = 'C:\ws\vault'; brain init --full --apply
$env:BRAIN_VAULT = 'C:\ws\vault'; brain search "welcome" --json

# 3. register the hourly maintenance task for this vault
.\scripts\install-brief-windows.ps1 -VaultPath C:\ws\vault
Known gap on Windows: the weekly brain-synthesis task is not auto-registered (only the hourly umbrella is). Everything else — capture, search, maintenance folds — is at parity.

3 · Fastest path: 3 commands inside Claude Code

/plugin marketplace add Autopsias/brainiac
/plugin install brainiac-manager@brainiac
/brainiac-install ~/my-brain

The manager plugin's install skill does everything §1 does plus the surrounding lifecycle: engine install, vault init + index, the hourly maintenance task, audit-key provisioning in your OS keychain, and the workspace registry entry. Later: /brainiac-update, /brainiac-health, /brainiac-uninstall.

4 · Existing machine → another project (second vault, third, …)

The install is per machine; vaults are per project. You never reinstall — you point the same brain at a different folder, and each vault automatically gets its own index and audit chain:

4a · New (empty) project vault

export BRAIN_VAULT=~/vaults/new-project
brain init --full --apply        # scaffold + seed + index — searchable immediately

4b · A folder that ALREADY has notes

export BRAIN_VAULT=~/vaults/existing-notes
brain init --full --apply        # scaffolds/validates — but SKIPS indexing (vault not empty)
brain rebuild                    # REQUIRED once: index the existing notes
The one gotcha worth memorizing: init --apply seeds-and-indexes only an empty vault. Point it at a folder that already contains notes and it deliberately won't touch them — so the index starts empty and the first search returns nothing until you run brain rebuild once.

4c · Import an external notes folder (Obsidian export, etc.)

brain init --full --import-from ~/ObsidianVault --yes   # host-only; stages, ingests, signs
brain rebuild                                            # if importing into a non-empty vault

Without --yes it shows a dry-run manifest and asks before ingesting. Imported notes without a classification: label are invisible to search until classified (deny-by-default) — mass-classification is part of a real migration, not polish.

5 · The embedding model — when the ~465 MB download happens

Brainiac separates “installed and searchable” from “semantic search at full quality”, so installation never blocks on a big download:

StageWhat runsNetwork
install + initindex built with the offline placeholder embedder (hash) — lexical search worksnone
brain warmup (or first real semantic use)downloads multilingual-e5-small (ONNX, ~465 MB, one-time, cached)Hugging Face, once
brain sync after warmupre-embeds the index with the real modelnone
brain warmup          # explicit one-time download (progress on stderr)
brain sync            # re-embed the existing index with the real model
brain status          # shows embedder: ready | pending

Air-gapped? Stay on BRAIN_EMBEDDER=hash (lexical only, zero downloads) or point $BRAIN_MODEL_CACHE at a pre-staged model directory.

6 · Verify, update, uninstall

brain doctor            # install + registry health check (--json for machines)
brain status            # index stats, snapshot age, pending drafts, embedder state
brain --help            # the always-current command contract

# update
bash /tmp/brainiac-install.sh     # re-run the installer, or in Claude Code: /brainiac-update
brain update --dry-run            # see what would change first

# uninstall (Claude Code)
/brainiac-uninstall               # removes engine, tasks, wiring — never your vault

Whichever channel installed the engine, wiring an AI client to a vault is always the same one command afterwards:

brain connect claude-code      # or: claude-desktop | codex | gemini
brain connect claude-desktop --remove   # undo

Claude Desktop's chat tab (the one surface that can't run a shell) uses the .mcpb bundle (dist/brainiac.mcpb) — a thin bridge that spawns the host's read-only brain-mcp; it never bundles a second engine.
