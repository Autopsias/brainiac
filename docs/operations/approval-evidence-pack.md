## SBOM note — build-env vs runtime (honest framing)

`sbom.cdx.json` is the **build-environment** SBOM (the `.venv` that froze the
artifact), generated by `cyclonedx-py environment`. It is a **superset** of what
ships: it lists build-only tools (PyInstaller's `altgraph`, `cyclonedx-bom`) and
the S05 eval extras (`ranx`, `numpy`, `fastparquet`). For a production release,
freeze from a **minimal locked venv** (only `[vec,audit,embed,yaml]` extras) so
the SBOM == shipped components; the runbook in `packaging/README.md` shows how.
The root component is correctly `profile-a-brain 0.2.0` and the format is
CycloneDX 1.5 (ingestible by Dependency-Track / GUAC).
